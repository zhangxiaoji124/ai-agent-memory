# amio 架构与实现技术细节（研讨文档）

> 面向 Agent 记忆的磁盘 HNSW 向量检索引擎 `amio`（CMake 工程 `agent-memory-io-cpp`）。
> 本文逐模块、逐文件讲清**实现精髓**——数据结构、算法、关键函数与设计取舍，供组内与老师研讨。
> 关联：`README.md`、`docs/实验报告-IO优化-2026-05-28.docx`、`docs/提升方案-基于实测结果.md`、`docs/Agent长时记忆-实现状态对照.md`。

## 0. 一页纸总览

**目标**：把 HNSW 图索引下沉到磁盘，在内存远小于数据集时仍能低延迟检索。核心矛盾是"图大装不下内存、检索却要快"，所以全栈都在做一件事——**少读盘、读得巧**。

**端到端数据流**：

```
build：向量 → HnswIndex 建图 → 写成磁盘文件（头 4KB + 每节点一个块）
 open：读文件头 → 据内存预算分三池 → BFS 把导航枢纽 pin 进静态缓存
search_disk：入口点出发 → 上层贪心导航 → layer0 beam search
            每访问一个节点 → 缓存命中?→否则读盘 → 算距离 → 扩展邻居 → 顺带异步预取邻居
```

**分层与目录**：

| 层 | 目录 | 职责 |
|----|------|------|
| 存储层 | `index/storage_layout`、`index/external_vectors`、`dataset/vector_dataset` | 磁盘文件布局、节点块读写、高维外置向量、原始数据集 mmap |
| 图算法层 | `index/hnsw`、`dataset/index_builder`、`dataset/index_build_common` | HNSW 建图（RobustPrune）、写盘格式化 |
| 检索路径 | `vector_store`（`search_disk`/`disk_search_layer`）、`util/simd_distance` | 磁盘 HNSW 搜索、SIMD 距离 |
| 缓存层 | `cache/graph_aware_cache`、`cache/isvm_scorer` | 静态 pin + 动态 LRU、ISVM 驱逐、PAIC |
| 预取层 | `prefetch/topology_prefetcher`、`prefetch/io_uring_backend` | 拓扑感知异步预取、io_uring 后端 |
| 运行时 | `runtime/memory_partition`、`runtime/static_subgraph`、`runtime/graph_reorder` | 内存分区 M0–M4、枢纽 pin、图重排 |
| 写链路 | `write/memtable`、`wal`、`nvtable`、`compaction`、`index_merger`、`remap_com`、`noblsm_commit` | LSM 式写入与合并 |
| 策略 | `policy/agent_io_policy`、`learning/` | builtin vs learned I/O 策略 |
| 评测 | `tools/eval_disk` | recall/QPS/延迟/磁盘读，多模式对比 |

## 1. 存储层

### 1.1 `include/index/storage_layout.h`（+ `.cpp`）——磁盘文件布局
**精髓**：定长块寻址，节点位置由 id 直接算出，无需任何索引表。

- **文件结构**：`[文件头 kHeaderSize=4096B]` 之后是 `N` 个定长节点块，每块 `kBlockSize`。
- **`IndexFileHeader`（4096B）**：`magic`、`version`、`total_nodes`、`max_layer`、`m`、`ef_construction`、`dim`、`entry_point`、以及 v2 外置向量区的 `vector_section_offset / vector_stride_bytes / vector_encoding`。
- **`NodeBlock`（v3 瘦身后 2048B）**：一个节点的"名片"——
  - `node_id`、`layer`、`neighbor_counts[8]`（每层邻居数）、`vec_dim`；
  - `vector[128]`（dim≤128 时内联向量，512B）；
  - `neighbors[8][32]`（8 层、每层至多 32 个邻居 id，1024B）；
  - `_padding` 补齐到 2048。
- **`node_offset(id) = kHeaderSize + id*kBlockSize`**：纯函数寻址，**这是"瘦身"的关键前提**——邻居的磁盘 offset 能现算，所以不必落盘（原 v1/v2 存了 `neighbor_offsets[8][32]`=2KB，本轮删除，块 4KB→2KB）。
- **`IndexFile`**：`pread_node/pwrite_node` 用 `node_offset()`+`sizeof(NodeBlock)` 读写，因此对块大小变化"自动正确"。读写用循环 `pread/pwrite`（`storage_layout.cpp`），跨平台（Win 用 `_lseeki64+_read`）。

### 1.2 `include/index/external_vectors.h`（+ `.cpp`）——高维向量外置区（v2）
**精髓**：dim>128（如 GIST 960 维）时向量塞不进 4KB 块，单独连续存在文件尾部，mmap 读取。

- `header_uses_external_vectors(h)`：`version≥2 && vector_section_offset≠0`。
- `default_vector_section_offset(total) = kHeaderSize + total*kBlockSize`（紧跟所有节点块之后）。
- `ExternalVectorStore`：`attach()` 把向量区 mmap 进内存；`l2_sq(query, node_id)` 直接在 mmap 上算距离（**已接 SIMD**）；支持 `Float32` 与 `UInt8`（bvec）两种编码。
- `write_vector_section()`：建索引时把每条向量按 `stride` 顺序写入尾部并回填 header（用 `std::function` 回调取向量）。

### 1.3 `src/dataset/vector_dataset.cpp`——原始数据集统一抽象
**精髓**：fvecs/bvecs 任意 dim 都 **mmap + 按需取子集**，不整文件载入。

- `open()` 探测 dim/数量/`record_bytes`，`map_file()` 做只读 mmap（Win:`CreateFileMapping`，Linux:`mmap`）。
- `get_float(i)` 把第 i 条记录转成 `float`（uint8 自动提升）；`load_subset_float(path, max, out)` 取前 N 条——bigann 子集就是这么来的。

## 2. 图算法层

### 2.1 `include/index/hnsw.h` + `src/index/hnsw.cpp`——HNSW 建图与内存检索
**精髓**：多层小世界图 + DiskANN 的 **RobustPrune** 多样化剪枝；Layer0 用 `2*m` 邻居。

- **数据结构**：`vectors_`（全部向量）、`node_level_`（每节点最高层）、`layer_neighbors_[layer][id]`（邻接表）。最多 8 层。
- **`insert(id, vec)`**：随机层高 → 从入口逐层贪心找近邻 → 每层 `select_neighbors` 选边 → `add_edge` 双向连边。
- **`robust_prune(center, candidates, cap)`**：按到 center 距离排序，依次选入；**遮挡判据 `alpha*d(已选,候选) ≤ d(center,候选)` 则剔除该候选**（保证多样性，避免邻居挤在一个方向）。
  - ⚠️ **本轮修复的召回 bug 就在这**：原代码把不等式写反（`d(已选,候选) ≤ alpha*d(center,候选)`），导致选入第一个邻居后几乎剪光所有候选，每节点退化到 ~1 个邻居，召回崩（SIFT 0.12 / GIST 0.007）。修正后恢复（SIFT 1.0 / GIST 0.999）。
- **`add_edge`**：邻居表**未满直接追加，满了才重跑 RobustPrune**（hnswlib/DiskANN 同款）。
  - ⚠️ **本轮建图加速**：原实现每条边都重剪枝（O(cap²)），密集图下 1M 建图不可用；改为"未满追加"后建图成本大降。
- **`l2_sq(a,b)`**：L2 平方距离，**已接 SIMD**（见 2.3）。
- **`search(query, k, ef)`**：内存态 best-first，用两个堆（候选小顶堆 + topk 大顶堆）+ `visited` 集。

### 2.2 `src/dataset/index_build_common.cpp`——把内存图写成磁盘块
**精髓**：`HnswIndex` 的邻接表 → `NodeBlock` 的格式化落盘。

- `make_node_block(idx, local_id, ...)`：填 `vector`（dim≤128 内联）、`neighbor_counts`、`neighbors`（带全局 id 偏移）。
  - 本轮起**不再写 `neighbor_offsets`**（v3 瘦身：offset 由 `node_offset(nid)` 现算）。
- 高维（v2）时调用 `external_vectors` 把向量追加到尾部区。

### 2.3 `include/util/simd_distance.h`——SIMD 距离（本轮新增）
**精髓**：一份头文件、运行时派发、跨平台兜底。

- `l2_sq_f32(a, b, n)`：x86 + GCC/Clang 用 `__attribute__((target("avx2,fma")))` 的 8 宽 AVX2+FMA 实现，`__builtin_cpu_supports("avx2")` 运行时选择；否则标量兜底。
- `AMIO_FORCE_SCALAR=1` 可强制标量，便于做 A/B。已接入 `HnswIndex::l2_sq`、`ExternalVectorStore::l2_sq`、`VectorStore::l2_sq(_block)`。
- 实测：GIST 960 维端到端 **2.31×**，SIFT 128 维 1.43×，召回不变。

## 3. 检索路径

### 3.1 `src/vector_store.cpp`——`search_disk` / `disk_search_layer`
**精髓**：在磁盘上**逐块复现** HNSW 搜索，并融合 θ 阶段切换与预取。

- **`search_disk(query, k)`**：
  1. 上层导航（layer = max_layer→1）：每层 `disk_search_layer(ef=1)` 贪心取最近点作为下一层入口（**不启用 θ**）。
  2. Layer0 beam search：`disk_search_layer(ef=ef_search)`，**启用 θ 阶段切换**。
  3. 可选与 MemTable 暴力结果归并（`merge_top_k`）。
- **`disk_search_layer(...)`**：候选小顶堆 + topk 大顶堆 + `visited`；每弹出一个节点：
  - `cache.get_or_load()` 取块（命中走内存，否则 io_uring 同步读或 pread）；
  - `prefetch.on_visit_node()` 触发邻居预取；
  - 用 `ext_vecs->l2_sq`（v2）或 `l2_sq_block`（v1 内联）算距离，扩展未访问邻居。
- **θ 阶段切换（GoVector 思想）**：当 `visited_count ≥ partition_theta * ef` 时进入 `phase2`（探索期），预取改用更大的 layer0 扇出。`io_metrics.theta_phase_switches` 可观测。

## 4. 缓存层（读优化核心）

### 4.1 `include/cache/graph_aware_cache.h` + `.cpp`——图感知双池缓存
**精髓**：静态 pin（永不淘汰的导航枢纽）+ 动态热点 LRU（ISVM 评分驱逐），并做 PAIC 预取追踪。本轮把它从"越大越慢"调成"O(1) 命中 + O(log n) 驱逐"。

- **两个池**：
  - `pinned_`（`unordered_map`）：静态常驻，命中即返回，**从不淘汰**；
  - `hot_`（`unordered_map<id, HotEntry>`）：动态热点，超出 `dynamic_capacity_bytes_` 触发淘汰。
- **`HotEntry`**：`block`、`tick`（LRU 时间戳）、`is_prefetch_loaded`（PAIC 标志）、`kv_group_bytes`（组大小）、`ev_key`（在驱逐索引中的键）。
- **准入控制（一次性访问排除）**：`cold_misses_[id]` 计数，第 `hot_insert_min_prior_misses` 次冷未命中才进 `hot_`（learned 默认 2）。**这是受限内存下学习策略能"装得下"的关键**。
- **驱逐索引 `ev_index_`（本轮新增，`std::set<(ev_key,id)>`）**：
  - ISVM 驱逐分 `score = tick_age/16 + C`，其中 `C`（预取标志、组大小）与时间无关；驱逐时 `tick_` 对所有条目相同，故 **argmax 等价于 argmax(16*C − tick)**，是个稳定键。
  - **O(log n) 驱逐**：取 `ev_index_` 最大键（`rbegin`）即最该淘汰者，保持 ISVM 排序不变（取代原 O(n) 全表扫描；32MB 缓存下 2174ms→17ms）。
  - **O(1) 命中**：命中时只更新 `tick`，**不动 `ev_index_`**；键变陈旧由**驱逐时惰性校正**（lazy priority queue：弹出候选→重算真实键→不符则回插重试）。修掉了"缓存越大命中越慢"的病（512MB 缓存 53→9.8ms，5.5×）。
- **PAIC 追踪**：`record_prefetch_submitted()` 记预取 id；命中带标志的条目 → `useful_prefetch_demand_hits++`；淘汰仍带标志 → `wasted_prefetch_evictions++`。
- **`get_or_load(file, uring, id, out)`**：未命中时优先 io_uring 同步读、否则 pread，计入 `disk_sync_block_reads`。

### 4.2 `include/cache/isvm_scorer.h` + `.cpp`——整数线性驱逐评分
**精髓**：固定权重的整数线性式，近似 Demand-MIN（**非训练 SVM**，是工程简化）。

- `eviction_score(is_prefetch_loaded, tick_age, kv_group_bytes)`：
  `= tick_age/16 + (is_prefetch_loaded? +64) + (组≤64B? −16 : 组≥512B? +24 : 0)`。
  分越高越该淘汰——即"预取未消费的、组大的、年龄大的"优先淘汰。

## 5. 预取层

### 5.1 `src/prefetch/topology_prefetcher.cpp`——拓扑感知预取
**精髓**：访问一个节点时，按策略把它的邻居块异步拉进来；可按磁盘 offset 排序以利顺序读。

- **`on_visit_node(node, layer, phase2)`**：先 `drain_completions_nonblocking(32)` 回收已完成 CQE（释放 slot），再按策略枚举邻居 id：
  - `use_layer_aware_prefetch`：只预取当前层邻居（减无效读）；否则全 8 层；
  - layer0 扇出受 `max_neighbor_fanout_layer0` 限制；
  - `sort_prefetch_by_disk_offset`：按 **`node_offset(nid)`**（本轮起现算，不再读块内字段）排序，偏向顺序读；
  - `submit_prefetch(ids)` 提交，并 `cache.record_prefetch_submitted(ids)`。
- `use_legacy_prefetch()`：策略等价于经典默认时走简化路径。

### 5.2 `src/prefetch/io_uring_backend.cpp`——异步 I/O 后端
**精髓**：固定大小的 SQ/slot 环，预分配对齐缓冲；无 io_uring 时退化为 noop。

- **`IoUringBackend`（Linux + `AMIO_HAS_URING`）**：`io_uring_queue_init(depth)`，`depth` 个 `Slot{buf, busy, node_id}`，buf 用 `posix_memalign(kBlockSize)` 对齐。
  - `submit_prefetch`：找空闲 slot → `io_uring_prep_read` 到 slot.buf → `sqe_set_data64(slot_idx)` → 批量 `io_uring_submit`（**非阻塞异步**）。
  - `drain_completions_nonblocking`：`peek_cqe` 取 `user_data=slot_idx` 释放 slot 复用。
  - `read_node_sync`：单块同步读（缓存未命中的快路径）。
- **`NoopBackend` / `#else` stub**：`submit_prefetch` 直接返回（不预取），`read_node_sync` 返回 false（回退 pread）。
  - ⚠️ **本评测机的 liburing 太旧缺 `io_uring_sqe_set_data64`，编译时 `AMIO_ENABLE_URING=OFF` → 走 stub**，所以异步预取收益在本机未体现。

## 6. 运行时（内存与布局）

### 6.1 `src/runtime/memory_partition.cpp`——内存分区器 M0–M4
**精髓**：按"数据类型 + 数据量/内存预算 ρ"自动选 profile，并切分三池、给出运行时超参。

- **分类 `classify()`**：
  - fvecs：data ≤ ram/2 → **M0 HOST_RESIDENT**；≤ ram*2 → **M1 HYBRID_FLOAT**；否则 **M2 DISK_FIRST**。
  - bvecs：≥400GB 且 ram∈[80,128]GB → **M4 BVEC_ULTRA**；≥100GB 或 ρ≥3 → **M3 BVEC_DISK_TIERED**；否则 M2/DISK_FIRST。
- **三池 `split_pool()`**：MemTable 10% / Static 20% / Dynamic 70%（HOST_RESIDENT 为 10/15/75）。
- **超参 `tune_profile_hints()`**：每 profile 给 `partition_theta`（M0=0、M1=0.30、M2+=0.35）、`ef_search_hint`、`prefetch_depth_hint`、距离核（fvec=`l2_f32`，bvec=`u8_l2`）、流式建图批大小。
- `resolve_ram_budget_bytes()`：`--ram-budget-gb` > 环境变量 `AMIO_RAM_BUDGET_GB` > 物理内存×80%。
- `apply_partition_to_config()`：把池大小写进 `Config`（`cache_size_mb`/`static_cache_mb`/`memtable_limit_mb`），并对 DISK_FIRST/HYBRID/BVEC profile 打开静态子图 pin。

### 6.2 `src/runtime/static_subgraph.cpp`——导航枢纽 pin
**精髓**：打开索引时从入口点 **BFS 展开上层（layer≥1）邻居**，把"高速路口"pin 进静态池，直到耗尽预算。

- `pin_static_navigation_subgraph(entry_point, max_layer, budget)`：队列 BFS，只扩展 layer≥1（导航层，非 layer0 海量底层），每访问一个节点 `cache.pin()`。**保证每次检索的导航起步全内存命中**。

### 6.3 `src/runtime/graph_reorder.cpp`——图重排（离线工具）
**精髓**：把拓扑邻近的节点在磁盘上排到相近 id，提升 layer0 局部性。

- `reorder_index_bfs`：从入口 BFS 分配新 id（拓扑近邻→物理近邻）。
- `reorder_index_gorder`：Gorder 风格——滑动窗口内最大化共享邻居数（VLDB'16 简化版，比 BFS 紧但慢 3–5×）。
- 重写所有块的 `node_id`/`neighbors`（v3 起不再写 offset）；v2 外置向量区同步重排。**仅 `tools/reorder_index.cpp` 离线调用，未接进 build/compaction；无 Porder（按访问频次加权）。**

## 7. 写链路（LSM 式）

> 评测主路径是磁盘**读**；写链路为骨架，且部分为**进程内模拟**（无真实 NVM/SSD/内核交互）。研讨时应明确区分。

| 文件 | 职责 | 成熟度 |
|------|------|--------|
| `write/memtable.*` | 内存写缓冲，满阈值 `drain()` 批量刷 | 部分（真实内存缓冲） |
| `write/wal.*` | 预写日志，崩溃恢复 | 部分（真实 append） |
| `write/nvtable.*` | DRAM `vector` 链，吸收 MemTable 批次 | **简化**：无 NVM、无 offset 数组，WAL 仍在跑（未达设计的 WAL-free） |
| `write/compaction.*` | 后台 `CompactionWorker` 线程 + 回调 | 部分（队列+回调，无完整 LSM 多层） |
| `write/index_merger.*` | `merge_batch_into_index` 把批次合并进索引文件 | 部分 |
| `write/remap_com.*` | UDB（未变更块）识别、记 remap 表 | **模拟**：无真实 `nvme_ioctl`/FTL 零拷贝 |
| `write/noblsm_commit.*` | check_commit/is_committed 免 fsync | **模拟**：用户态 grace 计时器，无内核/Ext4 协同 |

`VectorStore::insert` 流程：`wal->append` → `memtable.insert`；满 → drain 进 nvtable → 攒够批次 `submit_compaction_batch` → 后台 `merge_batch_into_index`。

## 8. 策略层与评测

### 8.1 `include/policy/agent_io_policy.h`——I/O 策略
**精髓**：一组可被 JSON 覆盖的运行时旋钮，调节预取与缓存准入。

- 字段：`prefetch_depth_upper/layer0`、`max_neighbor_fanout_layer0`、`use_layer_aware_prefetch`、`sort_prefetch_by_disk_offset`、`hot_insert_min_prior_misses`、`partition_theta`。
- `from_config()`：内置默认（builtin）；`merge_json_file()`：合并 learned JSON（learned）。
- `learning/train_agent_policy.py`：离线从轨迹/游走提 4 维特征（mean_fanout、std_fanout、seq_locality、high_layer_share）→ 规则选簇 → 导出两套参数 JSON。**离线/代理，非真实 Agent 遥测；簇内 sklearn 仅探针，不在线。**

### 8.2 `tools/eval_disk.cpp`——评测入口
**精髓**：同索引下做 builtin/learned、full/baseline 对比，输出全套 I/O 指标。

- 模式：`full`（缓存+预取+写框架）、`baseline1`（关缓存/预取/pin）、`baseline2`（仅磁盘 HNSW 主路径）。
- 关键开关：`--policy-mode`、`--rebuild`、`--recompute-gt`、`--ram-budget-gb`、`--memory-profile`、**`--cache-size-mb/--static-cache-mb`（本轮新增，做受限内存评测）**。
- 输出：recall@k、QPS、p50/p95/p99、`sum_disk_sync_block_reads`、cache 命中、预取有效/无效、`theta_phase_switches`；逐查询 CSV。
- ⚠️ 注意：`cache_size_mb=0` 在代码里意为"**无淘汰上限=无限缓存**"而非"无缓存"；`--recompute-gt 1` 对子集暴力重算 GT（本轮修了它此前 nq=0 跑 0 query 的 bug）。

## 11. 研讨可能被追问的点（建议提前准备答案）

1. **召回怎么保证？** RobustPrune 多样化建边 + 足够 ef_search；本轮修了遮挡不等式，SIFT/GIST 均恢复到 ~1.0/0.99。
2. **I/O 优化到底省在哪？** 缓存+静态 pin 让磁盘块读数砍半（BigANN 1M：208K→102K）；但**满内存下读本就便宜，优化要在"数据>内存、真读盘"时才回本**——届时 Linux 原生 pread 会 OOM/抖动，本引擎靠 pin+准入活下来。
3. **io_uring 收益？** 诚实回答：本机 noop 未体现；属待验证项（需升级 liburing）。
4. **ISVM 是真 SVM 吗？** 不是，是固定权重整数线性式（近似 Demand-MIN），属工程简化。
5. **写链路三件套（RemapCom/NobLSM/NVTable）真接 SSD/内核了吗？** 否，进程内模拟，定位为"机制验证"。
6. **瘦身为何安全？** 邻居 offset 是 `node_offset(id)` 的纯函数，落盘冗余；删除后块减半、缓存翻倍、I/O 减半，SIFT 实测 recall 不变。
