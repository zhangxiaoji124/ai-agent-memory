# 赛题剩余工作与交付计划

## 1. 赛题目标（整理版）

### 1.1 技术方向

面向 Agent 记忆场景，实现针对磁盘向量检索（HNSW + SSD）的 I/O 优化系统，核心包括：

- **读优化**：图感知缓存策略，减少随机读与 Page Cache 抖动。
- **预取优化**：利用异步 I/O（如 io_uring）与图拓扑预取，降低等待。
- **写优化**：借鉴 LSM 思想，处理高频增量写入对读延迟的影响（合并、压实、后台化）。

### 1.2 评审关注点

#### 优化策略（意见摘要）

1. **Agent 与机器学习**：结合 AI-agent 案例（如 Cursor、Qwen），学习访问模式与图索引结构，用机器学习 / 深度学习调节缓存参数与预取策略。
2. **异步读取**：按 I/O 与图访问特征区分数据量并测试；预取可先返回部分结果再异步补读；结合图拓扑做顺序化读取。
3. **Base cache**：软件缓存池与图访问顺序；LRU 或替代算法；命中走 CPU 侧缓存、未命中走 SSD；可结合学习策略与决策树做准入与替换。
4. **写优化**：随机写转为顺序追加写；借鉴 LSM 在 SSD 上的写路径优化，作为研究工作展开。

#### 相关期刊与会议

FAST、ATC、OSDI 等系统方向会议；VLDB、SIGMOD 等数据库方向会议。

#### 其他评价维度

- 性能指标（QPS、时延、召回）
- 创新性（基础功能上的优化深度）
- 代码质量
- 文档完整性

#### 2.3.3 各模块技术细节（实现级）

**（1）策略字段与 C++ 语义**

- `prefetch_depth_upper` / `prefetch_depth_layer0`：分别作用于 **layer>0**（高层贪心）与 **layer==0**（底层 best-first）阶段 `on_visit_node` 的预取重复深度。
- `max_neighbor_fanout_layer0`：限制 layer0 单次预取邻居上限（≤32）。
- `use_layer_aware_prefetch`：为真时以当前搜索层邻居为主，减少无关页预取。
- `sort_prefetch_by_disk_offset`：为真时按 `NodeBlock::neighbor_offsets[layer][i]` 排序再提交，偏向顺序读。
- `hot_insert_min_prior_misses`：`1` 表示每次冷加载即入 hot；`2` 表示第二次冷加载才入 hot（可能多一次磁盘读，换缓存洁净度）。

**（2）JSON 解析**

- 字符串查找 + `strtoul`、布尔前缀匹配；不解析任意嵌套 JSON。`_meta` 等扩展键由脚本置于后部，与解析键名冲突风险可控。

**（3）离线特征（Python）**

- 四维：`mean_fanout`、`std_fanout`、`seq_locality`、`high_layer_share`（见脚本 `FEATURE_DESCRIPTIONS_ZH` 与 `--dump-features`）。
- 当前为合成轨迹；接真实日志时需替换 / 扩展 `_synthetic_*_trace` 与特征构造。

**（4）行为兼容**

- 未配置 `agent_policy_path` 或文件打开失败：stderr 提示后使用 `from_config(cfg)`；预取在默认策略下与仅 `prefetch_depth` 的旧路径对齐（`TopologyPrefetcher::use_legacy_prefetch`）。
